## ✅ 체크리스트 항목

### 1️⃣ 발생 시간 확인

- [ ] 샘플 티켓에 기재된 발생 시간(UTC/KST)을 읽고 기록했는가?
- [ ] 샘플 로그의 타임스탬프와 티켓 발생 시간이 일치하는지 확인했는가?
- [ ] 시간대(Timezone) 오해 여부를 점검했는가? (UTC ↔ KST 변환 확인)

```
📌 기록란 (발생 시간):
  - 샘플 티켓 발생 시간 (UTC): _______________________
  - 샘플 로그 첫 CRC 오류 시간 (UTC): _______________________
  - 시간 차이: _______________________
```

---

### 2️⃣ 샘플 장비명 확인

- [ ] 샘플 티켓에 기재된 장비명(`SAMPLE_TOR_SW_01`)을 정확히 읽었는가?
- [ ] 장비 유형(TOR Switch — 샘플)을 확인했는가?
- [ ] 샘플 위치 정보(SAMPLE-RACK, SAMPLE-DATACENTER)를 기록했는가?

```
📌 기록란 (장비 정보):
  - 장비명 (샘플): _______________________
  - 장비 유형 (샘플): _______________________
  - 위치 (샘플): _______________________
```

> ❌ 실제 장비 시리얼, 실제 IP, 실제 관리 계정 정보를 이 란에 기입하지 마십시오.

---

### 3️⃣ 이벤트 내용 확인

- [ ] 샘플 로그에서 발생한 이벤트 유형(CRC 오류 증가, Link Down)을 확인했는가?
- [ ] 샘플 로그에 기재된 인터페이스(`Gi0/X`)를 읽었는가?
- [ ] 이벤트 발생 순서(CRC 증가 → 임계치 초과 → Link Down)를 파악했는가?

```
📌 기록란 (이벤트):
  - 이벤트 유형: _______________________
  - 영향 인터페이스 (샘플): _______________________
  - 이벤트 발생 순서: _______________________
```

---

### 4️⃣ 반복 여부 확인

- [ ] 샘플 티켓의 관찰 내용에서 반복 발생 횟수를 읽었는가?
- [ ] 샘플 시나리오 기준 재발 주기를 확인했는가?
- [ ] 반복 발생이 가정된 경우, 단일 이벤트와 다른 대응 시나리오가 있는지 학습했는가?

```
📌 기록란 (반복 여부):
  - 샘플 기준 발생 횟수: _______________________
  - 재발 주기 (샘플 가정): _______________________
  - 반복 여부 판단: 반복 □  단발 □  불명확 □
```

---

### 5️⃣ 티켓 상태 확인

- [ ] 샘플 티켓의 현재 상태(OPEN / INVESTIGATING 등)를 읽었는가?
- [ ] Escalation 필요 여부 항목을 확인했는가?
- [ ] 샘플 Escalation 대상 및 사유를 기록했는가?

```
📌 기록란 (티켓 상태):
  - 티켓 상태 (샘플): _______________________
  - Escalation 필요: 예 □  아니오 □
  - Escalation 사유 (샘플): _______________________
```

---

### 6️⃣ ⚠️ 원인 단정 금지

> 이 항목은 교육에서 매우 중요한 원칙을 연습하기 위한 항목입니다.

- [ ] **샘플 로그만 보고 원인을 단정하지 않았는가?**
  - ❌ 금지 예시: "케이블 불량이다", "포트가 고장났다" — 추가 데이터 없이 단정하는 행위
  - ✅ 권장 예시: "CRC 오류 패턴이 관찰되며, 케이블·포트·광모듈 문제 가능성을 포함하여 추가 확인이 필요함"
- [ ] 샘플 시나리오에서 가능한 원인 후보를 **복수(複數)**로 나열했는가?
- [ ] 추가 데이터 없이 조치를 선행하지 않았는가?

```
📌 기록란 (원인 후보 — 단정 금지):
  - 가능한 원인 후보 1 (샘플): _______________________
  - 가능한 원인 후보 2 (샘플): _______________________
  - 가능한 원인 후보 3 (샘플): _______________________
  - 다음 확인 필요 데이터 (샘플): _______________________
```
