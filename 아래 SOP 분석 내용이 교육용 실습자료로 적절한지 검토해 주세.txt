아래 SOP 분석 내용이 교육용 실습자료로 적절한지 검토해 주세요.

검토 기준:
1. 실제 AWS 내부 절차처럼 보이는 표현이 있는가?
2. 실제 장비 점검이나 실제 네트워크 진단을 요구하는 표현이 있는가?
3. 실제 티켓 번호, 실제 장비 시리얼, 실제 IP, 계정정보가 포함되어 있는가?
4. 장애 원인을 하나로 단정한 표현이 있는가?
5. 비전공자가 이해하기 어려운 표현이 있는가?

입력:
[이 교육용 샘플 SOP는 샘플 로그에서 Link Down 이벤트가 보였을 때, 학생이 다음 내용을 정리하는 연습을 하기 위한 문서입니다.

이벤트가 발생한 시간
관련된 샘플 장비명
같은 장비에서 반복 발생했는지 여부
Ticket opened 또는 Ticket escalated 같은 표현이 있는지 여부
Escalation이 필요한지 여부
Handover Memo 또는 Incident Report 초안에 넣을 간단한 기록

즉, 실제 장비를 점검하는 절차가 아니라 교육용 로그를 보고 관찰 내용을 정리하는 연습용 절차입니다.

절차 요약

먼저 샘플 로그에서 Link Down 이벤트가 표시된 시간을 확인합니다.
그다음 이벤트에 나온 SAMPLE, DEMO, EDU 접두어가 붙은 샘플 장비명을 확인합니다.

이후 같은 샘플 장비에서 비슷한 이벤트가 반복되는지 살펴보고, 로그 안에 Ticket opened 또는 Ticket escalated 같은 표현이 함께 있는지도 확인합니다.

중요한 점은 영향 범위나 원인을 단정하지 않는 것입니다. 로그에 실제로 보이는 내용만 정리해야 하며, 혼자 판단하기 어려운 내용은 Escalation 필요 여부로 표시합니다.

마지막으로 확인한 내용을 Handover Memo나 Incident Report 초안에 넣을 수 있도록 짧고 명확하게 정리합니다.

주의사항

이 SOP는 교육용 샘플 문서이므로 실제 장비에 접속하지 않습니다.

또한 실제 네트워크 진단 명령을 실행하지 않고, 실제 IP 주소, 실제 티켓 번호, 실제 장비 시리얼, 실제 계정정보도 사용하지 않습니다.]

출력 형식:
1. 수정이 필요한 표현
2. 수정 이유
3. 더 안전한 교육용 표현
4. 학생이 최종 확인해야 할 사항
