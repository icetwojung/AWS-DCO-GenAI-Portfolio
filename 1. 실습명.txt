Step 4. Git 설치 확인하기

터미널을 열고 아래 명령어를 실행합니다.

Windows PowerShell / macOS Terminal / Linux Terminal 공통
git --version

정상 예시는 아래와 비슷합니다.

git version 2.xx.x

Git이 설치되어 있지 않으면 아래 방식 중 본인 운영체제에 맞는 방법으로 설치합니다.

운영체제	설치 방법
Windows	Git 공식 Windows 설치 페이지에서 설치 파일을 받거나, 가능한 경우 winget install --id Git.Git -e --source winget 사용
macOS	Homebrew가 있으면 brew install git, 또는 xcode-select --install 사용
Linux	배포판 패키지 관리자 사용. 예: Ubuntu/Debian 계열은 sudo apt install git

Git 공식 설치 페이지는 Windows용 최신 설치 파일과 winget 설치 명령을 안내하고 있습니다. macOS의 경우 Git 공식 문서에서 Homebrew 설치 방식과 Xcode Command Line Tools의 xcode-select --install 방식을 안내합니다. Linux는 배포판 패키지 관리자를 사용하는 방식이 가장 쉽고, Debian/Ubuntu 계열은 apt-get install git, Fedora 계열은 dnf install git 방식이 안내되어 있습니다.

설치 후 터미널을 새로 열고 다시 확인합니다.

git --version
Step 5. VS Code 설치 확인하기

터미널에서 아래 명령어를 실행합니다.

code --version

정상적으로 설치되어 있으면 버전 정보가 표시됩니다.

code 명령어가 안 되더라도 VS Code 앱이 실행되면 일단 설치 확인으로 인정할 수 있습니다.

Windows에서 설치가 필요한 경우
공식 VS Code 다운로드 페이지에서 Windows User setup을 다운로드합니다.
설치 파일을 실행합니다.
설치 후 PowerShell을 닫았다가 다시 엽니다.
아래 명령어를 실행합니다.
code --version

Windows User setup은 관리자 권한이 필요하지 않아 대부분의 개인 계정 설치에 적합하고, 설치 후 콘솔을 다시 열면 code .로 폴더를 열 수 있습니다.

macOS에서 설치가 필요한 경우
공식 VS Code 다운로드 페이지에서 macOS용 .dmg 파일을 다운로드합니다.
.dmg 파일을 열고 Visual Studio Code.app을 Applications 폴더로 이동합니다.
VS Code를 실행합니다.
터미널에서 code 명령을 쓰려면 VS Code에서 Cmd + Shift + P를 누른 뒤 Shell Command: Install 'code' command in PATH를 실행합니다.
Terminal을 새로 열고 아래 명령어를 실행합니다.
code --version

macOS 공식 설정 안내에서도 VS Code를 Applications 폴더로 옮기고, Command Palette에서 Shell Command: Install 'code' command in PATH를 실행한 뒤 터미널을 다시 열면 code .를 사용할 수 있다고 설명합니다.

Linux에서 설치가 필요한 경우
공식 VS Code 다운로드 페이지에서 배포판에 맞는 .deb 또는 .rpm 파일을 다운로드합니다.
Ubuntu/Debian 계열은 .deb, Fedora/Red Hat 계열은 .rpm을 사용합니다.
설치 후 아래 명령어로 확인합니다.
code --version

VS Code 공식 다운로드 페이지에서는 macOS, Windows, Linux .deb, Linux .rpm 다운로드를 제공합니다.

Step 7. Node.js/npm 설치 여부 확인하기

13. 자주 발생하는 오류와 해결 방법
오류 상황	원인	해결 방법
git 명령어가 인식되지 않습니다.	Git이 설치되지 않았거나 PATH가 반영되지 않았을 수 있습니다.	Git 설치 후 터미널을 닫았다가 다시 열고 git --version을 실행합니다.

code 명령어가 실행되지 않습니다.	VS Code가 PATH에 등록되지 않았거나 터미널을 설치 전에 열어 둔 경우입니다.	VS Code를 직접 실행해 보고, Windows는 터미널을 다시 열고, macOS는 Command Palette에서 Shell Command: Install 'code' command in PATH를 실행합니다.
VS Code와 Visual Studio가 헷갈립니다.	이름이 비슷하지만 다른 도구입니다.	이 과정에서는 Visual Studio Code, 즉 VS Code를 사용합니다.
python --version이 안 됩니다.	Python이 설치되지 않았거나 명령어 이름이 다를 수 있습니다.	Windows는 py --version, macOS/Linux는 python3 --version도 시도합니다.
Python 설치 후에도 명령어가 안 됩니다.	PATH 설정이 반영되지 않았을 수 있습니다.	터미널을 다시 열고, 그래도 안 되면 오류 화면을 캡처해 Notion에 기록합니다.
node 또는 npm 명령어가 안 됩니다.	Node.js/npm이 설치되지 않았을 수 있습니다.	오늘은 필수 완료가 아니므로 “미설치 / 추가 확인 필요”로 기록합니다.
GitHub 로그인이 안 됩니다.	비밀번호 오류, 2단계 인증, 계정 미생성 등이 원인일 수 있습니다.	비밀번호나 인증 코드를 Notion에 적지 말고, 상태만 기록합니다.
GitHub 저장소 생성이 어렵습니다.	GitHub 화면이 낯설거나 계정 권한 문제가 있을 수 있습니다.	저장소 생성은 “준비 중”으로 기록하고, 화면 캡처 또는 질문 내용을 Notion에 남깁니다.
명령어를 어디에 입력해야 할지 모르겠습니다.	터미널 사용이 익숙하지 않을 수 있습니다.	Windows는 PowerShell, macOS는 Terminal을 사용합니다. VS Code가 열리면 VS Code Terminal을 사용해도 됩니다.
설치가 50분 안에 끝나지 않습니다.	네트워크 속도, 권한 문제, PC 환경 차이 때문입니다.	완료하지 못한 상태를 Notion에 기록하고, 오류 화면과 시도한 해결 방법을 제출합니다.
