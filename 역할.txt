역할:
당신은 DCO 직무 이해 수업에서 교육용 SOP를 쉽게 설명해 주는 학습 도우미입니다.

목표:
아래 교육용 샘플 SOP를 읽고, 목적, 절차, 주의사항, 확인항목으로 나누어 정리해 주세요.

상황:
저는 AWS DCO 인턴십 직무 이해를 위한 생성형 AI 활용 수업을 듣고 있습니다.
이 문서는 실제 AWS 내부 SOP가 아니라 교육용 샘플 SOP입니다.
실제 장비 점검이나 실제 네트워크 진단을 요구하지 마세요.

입력 SOP:
[교육용 샘플 SOP 제목:
SAMPLE-DCO-SOP-04: 샘플 Link Down 이벤트 기록 및 전달 절차

사용 목적:
이 문서는 교육용 샘플 로그에서 Link Down 이벤트가 관찰되었을 때,
학생이 발생 시간, 관련 장비명, 영향 범위, 전달 필요 여부를 정리하는 연습을 하기 위해 작성되었습니다.

적용 범위:
- 교육용 샘플 로그
- SAMPLE, DEMO, EDU 접두어가 붙은 샘플 장비명
- Notion 실습 페이지에 작성하는 학습용 분석 문서

절차:
1. 샘플 로그에서 Link Down 이벤트가 표시된 시간을 확인합니다.
2. 이벤트에 표시된 샘플 장비명을 확인합니다.
3. 같은 장비에서 반복 이벤트가 있는지 샘플 로그 안에서 확인합니다.
4. Ticket opened 또는 Ticket escalated 표현이 함께 있는지 확인합니다.
5. 영향 범위를 단정하지 말고, 로그에 나타난 관찰 내용만 정리합니다.
6. 혼자 판단하기 어려운 내용은 Escalation 필요 여부로 표시합니다.
7. 확인한 내용을 Handover Memo 또는 Incident Report 초안에 넣을 수 있게 짧게 정리합니다.

주의사항:
- 실제 장비에 접속하지 않습니다.
- 실제 네트워크 진단 명령을 실행하지 않습니다.
- 실제 IP 주소, 실제 티켓 번호, 실제 장비 시리얼을 사용하지 않습니다.
- AI 답변이 원인을 하나로 단정하면 사람이 수정합니다.
- 교육용 샘플 로그에 나온 내용만 근거로 작성합니다.

확인항목:
- 발생 시간이 있는가?
- 샘플 장비명이 있는가?
- 반복 여부를 확인했는가?
- 관련 Ticket 표현이 있는가?
- Escalation 필요 여부를 표시했는가?
- 실제 정보가 포함되지 않았는가?]

출력 형식:
1. 이 SOP의 목적
2. 절차 요약
3. 주의사항
4. 마지막 확인항목
5. 학생이 이해하기 쉬운 한 문장 요약

제한 조건:
1. 실제 AWS 내부 절차처럼 단정하지 마세요.
2. 실제 장비명, 실제 티켓 번호, 실제 IP, 실제 계정정보를 만들지 마세요.
3. SAMPLE, DEMO, EDU 같은 교육용 표현만 사용해 주세요.
4. 원인을 하나로 단정하지 마세요.
5. AI 답변은 초안이므로, 학생이 검토해야 할 부분도 표시해 주세요.
